Most problems with luci-app-xray fall into a few categories: missing asset files, configuration generation failures, DNS or dnsmasq integration issues, and transparent proxy not intercepting traffic. This page walks through each category with concrete diagnostic commands and fixes.

## Compatibility requirements

<Warning>
luci-app-xray **does not support** the following platforms:

- **LEDE / OpenWrt prior to 22.03** — requires firewall4 (nftables) and client-side rendering LuCI
- **Lean's OpenWrt Source** — ships a variant of LuCI based on OpenWrt 18.06

For these platforms, use Passwall or a similar project instead.
</Warning>

## Missing geodata asset files

**Symptom:** Xray logs the following warning and GeoIP/GeoSite routing rules have no effect:

```text
WARNING: at least one of asset files (geoip.dat, geosite.dat) is not found under /usr/share/xray.
```

**Fix:**

```bash
opkg update && opkg install v2ray-geoip v2ray-geosite
```

If the packages are unavailable in your feed, see [issue #52](https://github.com/yichya/luci-app-xray/issues/52#issuecomment-856059905) for alternative approaches.

## Service won't start

<Steps>
  <Step title="Check the system log">
    ```bash
    logread | grep xray
    ```
    Look for config generation errors or procd failure messages.
  </Step>
  <Step title="Verify the Xray binary is present">
    ```bash
    which xray
    ```
    If the binary is in a non-standard location, set the `xray_bin` option in the general UCI section.
  </Step>
  <Step title="Inspect the generated config">
    The init script writes the configuration to `/var/etc/xray/config.json` (up to 5 attempts). If the file is empty or missing, the config generation step failed.

    ```bash
    cat /var/etc/xray/config.json
    ```

    Test whether the config is valid:

    ```bash
    xray run -test -confdir /var/etc/xray/
    ```
  </Step>
</Steps>

## LuCI page not loading

Clear the LuCI module and index caches, then reload:

```bash
rm -rf /tmp/luci-indexcache* /tmp/luci-modulecache
luci-reload
```

If the page still fails to load, reboot the router.

## Transparent proxy not working

<Steps>
  <Step title="Check the nftables ruleset">
    ```bash
    nft list ruleset | grep xray
    ```
    If no xray rules appear, the firewall setup step failed. Check `logread | grep xray` for errors from the init script.
  </Step>
  <Step title="Verify IP routing policy rules">
    Xray's TProxy setup adds a routing policy rule for fwmark `0xfb` (251). Confirm it is present:

    ```bash
    ip rule show
    ```

    Expected output includes:

    ```text
    32765: from all fwmark 0xfb lookup 251
    ```

    For IPv6:

    ```bash
    ip -6 rule show
    ```
  </Step>
  <Step title="Confirm Xray is running">
    ```bash
    ps | grep xray
    ```
  </Step>
  <Step title="Check the tproxy interface selection">
    In the LuCI interface, verify that the correct LAN interfaces are selected in the transparent proxy settings. After changing interface selections, restart the service:

    ```bash
    /etc/init.d/xray_core restart
    ```
  </Step>
</Steps>

## DNS issues

### DNS not resolving correctly

Check the dnsmasq configuration file generated by Xray:

```bash
cat /tmp/dnsmasq.d/xray.conf
```

Verify the DNS server settings in the general UCI config match your intended upstream resolvers.

### dnsmasq not restarting after interface changes

If you are running dnsmasq v2.87 and DNS rules are not being applied after network interface changes, enable the `PACKAGE_XRAY_RESTART_DNSMASQ_ON_IFACE_CHANGE` build option when compiling the package.

## REALITY support

REALITY support is experimental. The protocol specification is not yet finalised and breaking changes may be introduced without notice. If REALITY stops working after an update, check the changelog for configuration format changes.

## Config debugging reference

| Task | Command |
|---|---|
| View generated config | `cat /var/etc/xray/config.json` |
| Validate config syntax | `xray run -test -confdir /var/etc/xray/` |
| View Xray log output | `logread \| grep xray` |
| Stream logs live | `logread -f \| grep xray` |
| Check firewall rules | `nft list ruleset \| grep xray` |
| Check routing policy | `ip rule show` |
| Check running processes | `ps \| grep xray` |

## Deprecated features

### Global SOCKS and HTTP inbounds

The global SOCKS inbound (port 1080) and HTTP inbound (port 1081) that listen on `0.0.0.0` are deprecated and will be removed in version 4.0.0. These settings have been moved to the preview app.

Use **Extra Inbound** to add replacement inbound ports. Avoid common ports like 1080 and set the listen address to a specific interface rather than `0.0.0.0`.

### Sniffing

Global sniffing settings are deprecated. Use FakeDNS instead to route traffic based on domain names. Sniffing may be reimplemented in a future version with a different interface.

### Global custom settings

The `custom_config` UCI option is deprecated and will be removed in version 4.0.0. Migrate to the [custom configuration hook](/operations/custom-configuration-hook) instead.

## Enabling preview features

Some deprecated or unstable settings are available in the **Xray (preview)** app:

<Steps>
  <Step title="Enable preview mode">
    In the LuCI interface, go to **Services → Xray → Extra Settings** and select **Preview or Deprecated**.
  </Step>
  <Step title="Reboot">
    Reboot your router. A new **Services → Xray (preview)** menu item will appear.
  </Step>
</Steps>
